Four-Phase Architecture
PHASE 01
Data & Features
Multi-source data ingestion with disk caching. 88 features per stock: technical indicators, macro regime, NLP sentiment via FinBERT. Rolling z-score normalization with zero look-ahead bias.
Polygon.io Yahoo Finance Finnhub FinBERT TimescaleDB
PHASE 02
RL Training
PPO with MLP baseline or Transformer policy. Per-stock temporal encoder + cross-asset attention. 4-stage curriculum learning. Optuna hyperparameter search. Walk-forward validation.
PPO Transformer Curriculum Optuna MLflow
PHASE 03
Live Execution
Alpaca paper and live trading. 7-layer risk management. FastAPI inference server with 15 endpoints. Prometheus + Grafana monitoring. Docker Compose one-command deployment.
Alpaca API FastAPI Prometheus Grafana Docker
PHASE 04
Autonomous Ops
CUSUM + KS-test drift detection. Champion/Challenger retraining with Welch's t-test. UCB1 ensemble voting. 7 scheduled jobs. Slack, Email & webhook alerts.
CUSUM Drift Ensemble UCB1 APScheduler Slack
Key Capabilities
🧠
Transformer Policy
Per-stock CLS-token temporal encoder + multi-head cross-asset attention. Learns inter-stock correlations implicitly.
📈
Realistic Backtest
Adjusted OHLCV, 5bps commission, sqrt(size/ADV) market impact, bid-ask spread. No look-ahead bias in features.
🎓
Curriculum Learning
4 stages from calm bull markets to full crisis. Advances only when Sharpe threshold is met on 3 consecutive evals.
🛡️
7-Layer Risk Control
Portfolio halt, daily halt, VIX scaling, position caps, sector limits, liquidity filter, Kelly sizing — independent of agent.
🔄
Auto-Retraining
Champion/Challenger with Welch's t-test. New model must beat old by Δ Sharpe ≥ 0.05 at p < 0.10 to be promoted.
🎲
UCB1 Ensemble
Multi-armed bandit weighting. Disagreement-aware position scaling: high model variance → reduced position size.
